3. Express App has it's own event listeners which can be defined using `app.on()` method, where the first argument will the event name and the 2nd argument will the callback to be executed to handle the triggered event.
4. In Node.js runtime environment, we're provided with `process` object, which is the reference to the current running process in Node.js. This `process` object has a method called `exit()` which syncrhonously exits the process, irrespective of whether or not there are any pending asynchronous process(s) in the event loop or not. The `exit()` takes a status code (number type) as an argument which gets printed on the terminal when the process is exited. Normally, upon completion of every process in Node.js this exit code is printed on the terminal, but this `exit()` method allows us to do it manually and syncrhonously at any point of time in the program/process. If we don't pass any status code as arg to the `exit()` then the default "success code" is `0` or the whatever value is of the `process.exitCode` property, if set. Mostly, we don't use `process.exit()` in programs, especially asynchronous operations, rather we just set the `process.exitCode` property, and let the operation to be completed normally and exited with the set exit-status-code.
5. Importing `dotenv` and configuring it in "ES Modules (Module JS)" file organizing system, which is a modern way in both browsers and modern Node.js. Also, learnt about passing `dotenv/config` by using `-r` flag on [ClipNexus's](https://github.com/VinayNoogler000/ClipNexus-YT-Clone/blob/main) `dev` script (mentioned in [package.json](https://github.com/VinayNoogler000/ClipNexus-YT-Clone/blob/main/package.json)) as `experimental-json-modules`.
6. Instlallation and Configuration of `CORS Middleware` by using its almost all options, for security and performance improvement. Also, learnt that using this middleware doesn't blocks Client-HTTP-Requests from disallowed origins, rather it clearly tells the browser whether the server's responses can be read by javascript (for allowed origins), or cannot read (for disallowed origins), which means the server will still be able to receive and respond to all the requests. So, in other words we can say that `CORS Policy` is browser's way of protecting the data in server's response from malicious activities by the user/client (who sent the request), and also, kinda, protecting the user (or its device) from malicious data in server's response.
7. We can limit the size of data we want in JSON or URL-Encoded form in the HTTP-Request by using `{limit: [size]}` and passing it as an argument to the `app.use(express.json({limit: "16kb"}))`. Also learnt that when we pass `{extended: true}` to the `app.use(express.urlencoded([➡️here⬅️]))`, then it just means that data in URL which was encoded to JS object can now also consists of Nested Objects. At last, to parse Cookies and perform CRUD ops. in it we use `cookie-parser` library.
8. In any MongoDB collection, if we want to search documents based of its specific properties, like `userName` property of documents/objects in `users` collections, then we set the `index: true` property when creating model using mongoose, to optimize the searching functionality. But, setting the property `index` to `true` is resource heavy, so we should be careful about it, and only set for those properties of documents/objects in collections, which we're going to use often for searching the documents/objects, else ignore it, as the searching operation can still be performed.
9. For Pagination in MongoDB, there's a plugin for mongoose called `mongoose-aggregate-paginate-v2` which can be used to perform pagination in MongoDB, and it also provides a lot of options for customization. 
10. For encrypting the password of users, we can use `bcrypt` library which provides a method called `hash()` which takes the password and a salt (number of rounds) as arguments and returns the hashed password. And to compare the hashed password with the plain text password, we can use `compare()` method of `bcrypt` library.
11. For generating JWT tokens, we can use `jsonwebtoken` library which provides a method called `sign()` which takes the payload (data to be stored in token), secret key and options as arguments and returns the generated token. And to verify the token, we can use `verify()` method of `jsonwebtoken` library.
